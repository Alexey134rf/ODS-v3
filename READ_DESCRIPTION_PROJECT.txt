Алгоритм работы парсера информационных донесённый от МЧС
*Команды выполняются из корневой директории проекта 
*Пути в скриптах используется абсолютные, возможно будет нужно отредактировать: copy_web_project_to_web_server.sh и copy_web_project_to_web_server.sh
Общая информация по проекту:
1)	Расположение скрипта парсера в контейнере: /root/ods_v3_lxc/lxc_project/ODS_linux.py
Команда для его запуска с Docker host:
CLI>docker exec parser_ods_v3_lxc /bin/sh -c "python3 /root/ods_v3_lxc/lxc_project/ODS_linux.py"
2)	Скрипт по копированию файлов и директорий, которые относятся к web-проекту: copy_web_project_to_web_server.sh
3)	Скрипт для выполнения резервного копирования проекта: backup_project.sh


1.	Сборка образа (возможны два варианта использования)
1.1.	Необходимо собрать docker образ с именем ods_parser_v3
CLI>docker build -t ods_parser_v3 .
Процесс сборки занимает определенное время, это нормально. Проверить результат можно путем просмотра списка образов на Docker host:
CLI >docker images

1.2.	Или указать сборку образа непосредственно в docker-compose.yml.
Для этого нужно закомментировать: image: ods_parser_v3 и
раскомментировать: # build: .

2.	Произвести запуск сервиса (контейнера) в фоновом режиме (detached)
Сервис выполняет функции:
1)	Парсинг информационных донесений (PDF-файлы) с почты https://email.volganet.ru
2)	Создание web-проекта по демонстрации спарсенных  файлов

Запуск сервиса:
CLI >docker-compose up –d

Результат успешного запуска — это наличие в списке контейнера parser_ods_v3_lxc и его статуса Up.
CLI >docker-compose ps

3.	Настройка отображения web-проекта на Apache HTTP Server
С помощью скрипта copy_web_project_to_web_server.sh происходит копирование файлов и директорий, которые относятся к web-проекту.
Соответствующая команда настраивается в Cron.

4.	 Заведение задачи в Cron (планировщик задач Linux) для автоматического парсинга файлов с определённой периодичностью посредством контейнера parser_ods_v3_lxc и копирование файлов web-проекта на Apache HTTP Server.

Задачи в Cron:
# Перечень задач для работы парсера информационных донесений от МЧС с электронной почты
# 1. Запуск парсера (получение файлов с почты) и копирование файлов, относящихся к web-проекту, на Apache HTTP Server
# 5 */2 * * * запуск парсера каждые 2 часа в 5 минут
CLI>5 */2 * * * docker exec parser_ods_v3_lxc /bin/sh -c "python3 /root/ods_v3_lxc/lxc_project/ODS_linux.py" && /root/docker_dev/ods_v3/copy_web_project_to_web_server.sh
# 2. Создание резервных копий 
# 0 3 * * * запуск резервного копирования каждый день в 03:00. 
# Резервные копии хранятся в течении 7 даней
CLI>0 3 * * * /root/docker_dev/ods_v3/backup_project.sh
